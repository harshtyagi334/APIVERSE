# Security Policy

## Supported Versions

We provide security updates for actively maintained versions of this project. Older versions may not receive patches.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| 0.x     | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly:

- **Contact**: Send details to security@apiverse.dev (replace with your actual email).  
- **Acknowledgment**: We aim to respond within 48 hours of receiving your report.  
- **Updates**: You will receive weekly progress updates until the issue is resolved.  
- **Resolution**: Once fixed, we will publish a security advisory and, if you wish, credit you for the discovery.  
- **Disclosure**: Please do not publicly disclose the vulnerability until we have released a patch.  

## Scope

This policy applies to:
- Core application code (React, Vite, Firebase integration).  
- Configuration files and deployment scripts.  
- API endpoints and authentication flows.  

## Responsible Disclosure

We encourage private reporting of vulnerabilities. Please avoid posting issues publicly on GitHub until the fix is confirmed and released.
